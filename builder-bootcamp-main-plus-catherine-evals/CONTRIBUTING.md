# Contributing (minimal)

For now, keep contributions lightweight and focused on lab placeholders and scaffolds.

- Keep PRs small and focused on one change.
- Branch naming: `feature/<lab>-<short-desc>`.
- PR description (2–3 sentences): what/why, scope, how to verify.
- Coordinate lab guides with Sean before merging substantial content.
- Do not include private data or internal-only links.

Thank you for helping build Builder Bootcamp!
